Visual Studio Code - Erweiterungen in ein benutzerdefiniertes Verzeichnis laden Datum: 2020-09-01 Betriebssystem: Windows 10 Enterprise 1909 Visual Studio Code: Version 1.50 Problembeschreibung: Auf meinem Arbeitsplatz ist ein strenges Rechtemanagment vorhanden, sodass ich nicht beliebige Programme an beliebigen Orten speichern kann. Glücklicherweise darf ich Visual Studio Code installieren, dessen Standardpfad für Erweiterungen sich leider nicht so einfach anpassen lässt. Im selben Verzeichnis wie VSC erstelle ich nun einen Ordner "VSCode_extensions". Hier gilt zu beachten, diesen nicht als Unterordner in den Installationspfad von VSC zu erstellen. Bei jedem automatischen Update würde dieser dann gelöscht (wie ich erfahren durfte).
